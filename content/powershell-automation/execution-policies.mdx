# PowerShell Execution Policies & Script Security

## Why this matters

One of the most common PowerShell errors is:

```
running scripts is disabled on this system
```

If you don’t understand execution policies, you won’t be able to:

- run scripts reliably
- debug script failures
- configure automation environments
- explain PowerShell security behavior

This comes up constantly in real-world Windows environments.

---

## Key Concepts

- Execution policies control **how PowerShell runs scripts**
- They are **not a security boundary**, but a safety feature

---

### Common Execution Policies

- **Restricted**
  - No scripts allowed
- **RemoteSigned**
  - Local scripts allowed
  - Downloaded scripts must be signed
- **AllSigned**
  - All scripts must be signed
- **Unrestricted**
  - All scripts allowed (with warnings)

---

### Scope Matters

Policies can be set at different levels:

- Process (temporary)
- CurrentUser
- LocalMachine

---

## Quick Summary

Execution policies determine whether PowerShell scripts are allowed to run and under what conditions.

---

## Quick Lab

### Step 1 — Check current policy

```powershell
Get-ExecutionPolicy
```

### Step 2 — See all scopes

```powershell
Get-ExecutionPolicy -List
```

### Step 3 — Set policy for current user

```powershell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```

---

## What to Notice

- Policies differ by scope
- Changing policy may require admin privileges
- Some environments enforce policies via group policy
- Errors often come from restrictive defaults

---

## Real-World Example

You try to run:

```powershell
.\script.ps1
```

And get:

```
running scripts is disabled on this system
```

Why?

Because the execution policy is set to `Restricted`.

---

## Debugging Strategy

When scripts won’t run:

1. Check execution policy:

```powershell
Get-ExecutionPolicy -List
```

2. Identify the active scope
3. Adjust policy if appropriate:

```powershell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```

4. If restricted by organization:
   - check group policy settings

---

## Interview Angle

You should be able to explain:

- Execution policies control script execution behavior
- They are not true security boundaries
- Policies can be set at different scopes
- Common issues come from restrictive defaults

Strong answer example:

“Execution policies determine whether scripts can run in PowerShell. They’re a safety feature rather than a strict security boundary. If a script won’t run, I check the current policy and scope, then adjust it appropriately or verify if group policy is enforcing it.”
